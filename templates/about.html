About BG Bank

BG Bank is a class demo banking system built to show how a secure distributed application can separate the web client, authentication service, and banking server into independent processes communicating via RPC.

The system uses XML-RPC for service communication, Fernet encryption for login tickets, SQLite for data storage, and rule-based fraud detection for suspicious activity.

Our Purpose

BG Bank demonstrates authentication, account management, encrypted tickets, transaction records, audit logs, and thread-safe banking in one project.

Security Focus

Login passwords and action passwords are stored as hashes. Sensitive actions like withdrawal and transfer require the action password before money can move.

Fraud Detection

Transactions are checked for large amounts, unusual activity, quick balance drops, daily transfer limit violations, and rapid transfers to different receivers.

Distributed Design

The browser only talks to Flask. Flask calls the Authentication Service (port 8001) and Banking Server (port 8002) through XML-RPC — a true RPC-based architecture.
