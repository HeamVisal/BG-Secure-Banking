Main project reviewed:
- Flask web app + XML-RPC auth service + XML-RPC banking service + SQLite.
- Important review findings from project review:
  1. docker-compose enables APP_LOG_SENSITIVE=1, and code can log password hashes, submitted password SHA256 fingerprints, full encrypted tickets, and registration payloads. This is a serious security risk.
  2. Deposit/withdraw/transfer writes are not atomic. Transfer updates balances and inserts transaction rows through separate committed DB calls. A crash/error can leave inconsistent balances/ledger.
  3. POST routes do not have CSRF protection.
  4. Registration creates user/profile/account in separate operations and does not roll back on partial failure.
  5. Money values use float/SQLite REAL; integer cents or Decimal would be more correct.
